<footer>
 <p>© 2023 Your Company</p>

  • <header>: Contains introductory content and navigation
  • <nav>: Navigation menu
  • <main>: The primary content area (only one per page)
  • <article>: Self-contained content
  • <section>: Thematic grouping of content
  • <aside>: Content tangentially related to the main content
  • <footer>: Footer information for the page
<picture>

 <source srcset="medium.jpg" media="(min-width: 768px)">
 <img src="small.jpg" alt="Descriptive text for accessibility" loading="lazy" width="800" height="600">
</picture> 		Responsive Images with Art Direction: The <picture> element allows serving different images based on viewport size or other criteria. The loading="lazy" attribute defers loading of off-screen images. Including width and height attributes prevents layout shifts while the image loads.
<form>
 <label for="email">Email:</label>
 <input type="email" id="email" name="email" autocomplete="email" required>

 <label for="password">Password:</label>
 <input type="password" id="password" name="password" autocomplete="current-password" required minlength="8">

 <div>
  <input type="checkbox" id="remember" name="remember">
  <label for="remember">Remember me</label>
 </div>

 <button type="submit">Sign In</button>
</form> 		Accessible Form Best Practices:
- Each input has an associated <label> with matching for and id attributes
- Using proper input types for validation (email, password)
- autocomplete attributes help browsers auto-fill information
- required attribute for form validation
- Using <button type="submit"> instead of <input type="submit"> for better styling options
<button aria-label="Close" class="close-btn">×</button> Accessible Close Button: The aria-label provides an accessible name for screen readers. Using the HTML entity × creates a proper "×" close symbol rather than the letter "X".
<div role="alert" aria-live="assertive" id="error-message" class="sr-only">
 Form submission failed. Please try again.
</div>

<div class="visually-hidden">Content only for screen readers</div> 		Accessibility for Dynamic Content: aria-live regions announce changes to screen reader users. Values include:
- assertive: Announces immediately, interrupting the current task
- polite: Announces when the user is idle

The visually-hidden or sr-only class hides content visually but keeps it accessible to screen readers.
<video controls crossorigin="anonymous" preload="metadata">
 <source src="video.mp4" type="video/mp4">
 <source src="video.webm" type="video/webm">
 <track kind="subtitles" src="captions.vtt" srclang="en" label="English">
 Your browser does not support the video element.
</video> 		Video with Accessible Captions: Modern approach to embedding video with multiple sources and text tracks.
- Multiple <source> elements for browser compatibility
- <track> element adds captions/subtitles for accessibility
- preload="metadata" loads only video dimensions and timeline
- Fallback text for browsers that don't support video
<details>
 <summary>Click to expand</summary>
 <p>Hidden content that will be revealed when the summary is clicked.</p>
</details> 		Collapsible Content: The <details> and <summary> elements create native accordions without JavaScript. This pattern is useful for FAQs, additional information, or any content that needs to be hidden initially.

FLEXBOX
Code Description
.flex-container {
 display: flex;
 /* or */
 display: inline-flex;
} 		Create a Flex Container: Establishes a flex formatting context for all its direct children. display: flex makes the container a block-level element, while display: inline-flex makes it inline.
.flex-container {
 display: flex;
 flex-direction: row; /* default */
 /* Or */
 flex-direction: row-reverse;
 flex-direction: column;
 flex-direction: column-reverse;
} 		Flex Direction: Defines the main axis of the flex container.
- row: Items flow horizontally (left to right)
- row-reverse: Items flow horizontally (right to left)
- column: Items flow vertically (top to bottom)
- column-reverse: Items flow vertically (bottom to top)
.flex-container {
 flex-wrap: nowrap; /* default */
 /* Or */
 flex-wrap: wrap;
 flex-wrap: wrap-reverse;
} 		Flex Wrap: Controls whether flex items are forced into a single line or can be wrapped onto multiple lines.
- nowrap: All items in a single line (may overflow)
- wrap: Items wrap onto multiple lines if needed
- wrap-reverse: Items wrap onto multiple lines in reverse direction
.flex-container {
 /* Shorthand for flex-direction and flex-wrap */
 flex-flow: row nowrap; /* default */
 /* Or */
 flex-flow: column wrap;
} 		Flex Flow: Shorthand property for flex-direction and flex-wrap. The first value specifies direction, and the second value specifies wrap.
.flex-container {
 justify-content: flex-start; /* default */
 /* Or */
 justify-content: flex-end;
 justify-content: center;
 justify-content: space-between;
 justify-content: space-around;
 justify-content: space-evenly;
} 		Justify Content: Aligns flex items along the main axis of the flex container.
- flex-start: Items are packed toward the start
- flex-end: Items are packed toward the end
- center: Items are centered
- space-between: Items are evenly distributed with the first item at the start and the last item at the end
- space-around: Items are evenly distributed with equal space around them
- space-evenly: Items are distributed so that the spacing between any two items is equal
.flex-container {
 align-items: stretch; /* default */
 /* Or */
 align-items: flex-start;
 align-items: flex-end;
 align-items: center;
 align-items: baseline;
} 		Align Items: Aligns flex items along the cross axis of the flex container.
- stretch: Items stretch to fill the container (default)
- flex-start: Items are aligned at the start of the cross axis
- flex-end: Items are aligned at the end of the cross axis
- center: Items are centered on the cross axis
- baseline: Items are aligned based on their baselines
.flex-container {
 align-content: stretch; /* default */
 /* Or */
 align-content: flex-start;
 align-content: flex-end;
 align-content: center;
 align-content: space-between;
 align-content: space-around;
} 		Align Content: Aligns flex lines when there is extra space in the cross axis. Only works when there are multiple lines (flex-wrap: wrap).
- stretch: Lines stretch to take up remaining space
- flex-start: Lines packed at the start
- flex-end: Lines packed at the end
- center: Lines centered
- space-between: Lines evenly distributed with first line at the start and last line at the end
- space-around: Lines evenly distributed with equal space around each line
.flex-container {
 gap: 10px; /* Equal gap in both directions */
 /* Or */
 gap: 10px 20px; /* row-gap column-gap */
 /* Or individually */
 row-gap: 10px;
 column-gap: 20px;
} 		Gap Property: Sets the spacing between flex items. Much cleaner than using margins, which can cause issues with edge items. Works in both flexbox and grid layouts.
- gap: Sets both row and column gaps
- row-gap: Sets spacing between rows
- column-gap: Sets spacing between columns
.flex-item {
 order: 0; /* default */
 /* Negative values allowed */
 order: -1; /* Comes before items with higher order */
 order: 1; /* Comes after items with lower order */
} 		Order: Controls the order in which flex items appear. Items with a lower order value appear first. Default is 0 for all items. Be careful with overuse, as it can create confusion between visual order and source order, potentially affecting accessibility.
.flex-item {
 flex-grow: 0; /* default */
 /* Or */
 flex-grow: 1; /* Share available space */
 flex-grow: 2; /* Takes twice as much space as items with flex-grow: 1 */
} 		Flex Grow: Defines how much a flex item can grow relative to other items if there's extra space. Default is 0 (item doesn't grow). The value represents the proportion of available space the item should take relative to other items.
.flex-item {
 flex-shrink: 1; /* default */
 /* Or */
 flex-shrink: 0; /* Won't shrink */
 flex-shrink: 2; /* Shrinks twice as much as default */
} 		Flex Shrink: Defines how much a flex item will shrink relative to other items when there isn't enough space. Default is 1 (item can shrink). Higher values mean the item will shrink more compared to items with lower values.
.flex-item {
 flex-basis: auto; /* default */
 /* Or */
 flex-basis: 200px; /* Initial size */
 flex-basis: 25%; /* Percentage of parent */
 flex-basis: 0; /* Distributes based solely on flex-grow */
} 		Flex Basis: Defines the initial size of a flex item before growing or shrinking. It sets the size of the item along the main axis.
- auto: Looks at the item's width or height properties
- content: Sizes based on the item's content
.flex-item {
 /* Shorthand for flex-grow, flex-shrink, and flex-basis */
 flex: 0 1 auto; /* default */
 /* Common presets */
 flex: 1; /* Same as flex: 1 1 0% */
 flex: auto; /* Same as flex: 1 1 auto */
 flex: none; /* Same as flex: 0 0 auto */
} 		Flex Shorthand: Combines flex-grow, flex-shrink, and flex-basis in one declaration.
- flex: 1: Item can grow and shrink, with a basis of 0%
- flex: auto: Item can grow and shrink, considering its size
- flex: none: Item can't grow or shrink (fixed size)
.flex-item {
 align-self: auto; /* default */
 /* Or */
 align-self: flex-start;
 align-self: flex-end;
 align-self: center;
 align-self: baseline;
 align-self: stretch;
} 		Align Self: Overrides the container's align-items property for individual flex items. This allows precise control over how specific items align along the cross axis.
/* Centering an item with flexbox */
.centered-container {
 display: flex;
 justify-content: center;
 align-items: center;
 min-height: 100vh; /* For full-height centering */
} 		Perfect Centering: One of the most powerful features of flexbox is the ability to easily center content both horizontally and vertically. This pattern is widely used for modal dialogs, card layouts, and hero sections.
/* Common flex layouts */
/* Equal width columns */
.equal-columns {
 display: flex;
 gap: 1rem;
}
.equal-columns > * {
 flex: 1;
}

/* Sidebar layout */
.sidebar-layout {
 display: flex;
 gap: 2rem;
}
.sidebar {
 flex: 0 0 250px;
}
.content {
 flex: 1;
} 		Common Flexbox Layouts:
- Equal width columns: Using flex: 1 on all children makes them grow to take equal space.

- Sidebar layout: A fixed-width sidebar (flex: 0 0 250px) with a flexible content area (flex: 1). The sidebar won't grow or shrink while the content area fills available space.
