animation shorthand property
animation-name   Specifies the name of the keyframe you want to bind to the selector

animation-iteration-count   Specifies how many times an animation should be played

animation-direction  Specifies whether or not the animation should play in reverse on alternate cycles

animation-fill-mode  Specifies what values are applied by the animation outside the time it is executing applies the settings of anaimation of the 0% before the animation starts vice versa with forwards

Handy tips and caveats

Code Description
Properties related to text are inherited , font-family, font-size, color, etc;

Properties related to length are usually not eg padding, margin

the computed value of a property is what gets inherited, not the declared value. Inheritance of a property only works if no one declares a value for that property,

the inherit keyword forces inheritance on a certain property.

the initial keyword resets a property to its initial value 		Inheritance
Z-index will not work on a if we have not changed the positioning from the default "static"; Z-index
1 .Normal flow
Default positioning scheme, Not floated, Not absolutely positioned, Elements laid out according to their source order
default -> position: static/relative;

2. Floats
Element is removed from the normal flow, Text and inline elements will be wrapped around the floated element
The container will not adjust its height to the element so we must use clear fix hack! and set overflow:auto;

3. Absolute positioning
Element is removed from the normal flow, No impact on surrounding content or elements, We use top,right,bottom, left to offset the element from its relatively placed container.
position:fixed, position:absolute; 		Positioning schemes
There is a distinction in using percentages for with fonts and with length or distance measures
Font:
When using percentages with font the reference is always the parent elements font size

Length: (padding, width, height, margin)
When using percentages with length elements the reference is always the parent elements width 		Percentages / How are units converted from relative to absolute?
The cascade happens first and goes in the following order, Importance->Specificity->Source order

Importance
  1. User !important declarations
  2. Author !important declarations
  3. Author declarations
  4. User declarations
  5. Default browser declarations

Specificity
  1. Inline styles
  2. IDs
  3. Classes, Pseudo classes and attributes
  4. Elements and Pseudo elements
Specificity is worked out the following way each time there is and occurrence then one gets added to the tally and whichever element has the highest specificity wins

A selector that contains 1 ID is more specific that one with 1000 classes
A selector that contains 1 class is more specific that one with 1000 elements
The universal selector * has no specificity value (0,0,0,0); (inline, IDs, classes, Elements) -> (0,0,0,0) 		How CSS is Parsed, The Cascade and Specificity
